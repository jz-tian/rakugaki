# 落書き — Rakugaki

**An AI-powered drawing game with a Japanese aesthetic.**

Get a random prompt, draw it in 120 seconds, and let Gemini Vision score your work. Clear the threshold to advance — or retry until you do.

---

## Features

### Drawing Canvas
- **Freehand drawing** via Pointer Events (mouse, stylus, touch)
- **Three brush styles** — Normal, Pencil, and Ink (each with distinct stroke rendering)
- **Flood fill** tool with click-to-fill, powered by a persistent off-screen canvas for performance
- **20-color palette** plus a custom color picker
- **Adjustable brush size** (1–30px) via a compact slider
- **Undo / Redo** with full history stack (snapshot saved on pointer-up, so redo always restores the completed stroke)
- **Clear canvas** with a single action
- Canvas is locked during loading and scoring phases to prevent accidental input

### AI Scoring
- Prompts are generated by **Gemini 2.5 Flash** at the start of each round, tailored to the selected difficulty and level number
- On submission, the canvas is exported as a PNG and sent to **Gemini Vision** alongside the original prompt
- The model returns a **0–100 score** and a short humorous comment
- Scoring is real visual understanding — the model reads the image and compares it to the prompt
- A **signed HMAC token** (SHA-256) ties each submission to its original prompt, preventing score manipulation

### Difficulty & Progression

| Difficulty | Style Band | Pass Threshold |
|---|---|---|
| Easy | Single universally recognisable objects — sun, apple, fish, star | 60 / 100 |
| Normal | Simple objects with one added element or gentle action — cat napping, dog with a ball | 75 / 100 |
| Hard | Short everyday scenes with 2–3 elements — person on a bicycle, child flying a kite | 85 / 100 |

- Level number increases each time you pass; a `levelTier()` function maps it to four complexity tiers (tiers 1–4) so the ramp is gentle — tier 1 stays at single-word concepts all the way through level 2
- Fail → retry the same level; Pass → advance to the next
- A **Reset to Lv.1** button appears on the home page whenever you are past level 1, so you can restart the progression without changing difficulty

### 120-Second Timer
- Every round is **120 seconds**, regardless of difficulty
- Displayed as an **enso-style circular ring** that depletes clockwise — powered by SVG `stroke-dashoffset` with a 1s linear transition per tick for a smooth sweep
- Ring and numeral turn vermillion with a gentle pulse animation at ≤ 20 seconds remaining
- Timer auto-submits when it reaches zero; the result page notes the time-out

### Give Up
- A **Give Up** button is always visible during drawing
- Requires confirmation before navigating back to the home page — no accidental exits

### Gallery
- Every completed round (pass or fail) is saved to **localStorage** (up to 50 works)
- The gallery on the home page shows all past works as cards with score stamps in hanko-seal style
- Click any card to **open a lightbox** with the drawing at full size, the prompt, score, and the AI's comment
- Works can be **deleted** from the lightbox with a two-step inline confirmation

### Internationalisation
- Full **English / 中文** toggle, persisted in localStorage
- All UI strings are loaded from `locales/en.json` and `locales/zh.json` at build time — no runtime fetch
- Prompts and AI comments are generated in the selected language
- The toolbar shows **Japanese labels** with English subtitles when the language is set to EN

### Design
- Japanese wabi-sabi aesthetic throughout — Shippori Mincho (Japanese serif), Cormorant Garamond (Western serif), warm washi-paper background
- Design tokens via CSS custom properties (`--beni`, `--ink`, `--ink-2`, `--ink-3`, `--rule`, `--surface`, `--bg`)
- Manuscript corner marks, dot-grid washi ground, enso timer ring, hanko score stamps

## Getting Started

### Prerequisites

- Node.js 18+
- A [Google AI Studio](https://aistudio.google.com/app/apikey) API key (free tier sufficient for normal use)

### Installation

```bash
git clone https://github.com/jz-tian/rakugaki.git
cd rakugaki
npm install
```

### Environment Variables

Copy the example and fill in the values:

```bash
cp .env.example .env.local
```

| Variable | Description |
|---|---|
| `GEMINI_API_KEY` | Google AI Studio API key |
| `PROMPT_SECRET` | Random secret for HMAC signing — generate with `openssl rand -hex 32` |

### Running Locally

```bash
npm run dev          # Start dev server at http://localhost:3000
npm run build        # Production build (runs ESLint + type check)
npm test             # Run all 33 tests
npm run test:watch   # Watch mode
npx tsc --noEmit     # Type check only
```

## API Reference

### `POST /api/generate-prompt`

Generates a drawing prompt and returns a signed token.

**Request**
```json
{
  "level": 1,
  "difficulty": "normal",
  "language": "zh"
}
```

**Response**
```json
{
  "prompt": "一只在打伞的猫",
  "token": "<base64url>.<hmac>"
}
```

Tokens expire after **10 minutes**. The score route will reject expired tokens with `400`.

---

### `POST /api/score`

Scores a drawing against its original prompt.

**Request**
```json
{
  "imageBase64": "data:image/png;base64,...",
  "promptToken": "<token from generate-prompt>",
  "difficulty": "normal",
  "language": "zh"
}
```

**Response**
```json
{
  "score": 78,
  "comment": "能认出来是猫，但那把伞画得更像是一个飞碟。",
  "passed": true
}
```

---

## Security Notes

- The prompt token format is `base64url(JSON.stringify({ prompt, difficulty, exp })) + "." + hmac_hex`
- `crypto.timingSafeEqual` is used for constant-time HMAC comparison — both sides are kept as `Buffer` (not hex strings) to ensure equal length
- The canvas image is sent to Gemini Vision from the server — it never touches third-party storage
- No user accounts or server-side state; all persistence is in the browser

## Testing

Jest is configured with **two separate projects** to avoid environment conflicts:

| Project | Environment | Paths |
|---|---|---|
| `node` | Node.js | `lib/__tests__/**`, `app/api/**/__tests__/**` |
| `jsdom` | jsdom | `components/__tests__/**` |

The Gemini client uses a factory pattern (`getGenAI()` called per-invocation) so Jest mocks apply correctly without module-level singleton issues.
